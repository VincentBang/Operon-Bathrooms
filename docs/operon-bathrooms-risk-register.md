# Operon Bathrooms Risk Register

| Risk                                         | Area                     | Current control                                                                  | Status     | Next action                                                                                                                          |
| -------------------------------------------- | ------------------------ | -------------------------------------------------------------------------------- | ---------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| Supabase contract drift                      | Supabase verification    | Approved Bathrooms project was re-verified on 2026-06-19; anon has insert-only estimate access and private tables remain service-role only. | Controlled | Re-run Supabase verification after every schema, grant or policy change. |
| Broad Supabase default anon grants           | Supabase security        | Anon grants were tightened on the approved Bathrooms project; only `bathroom_estimates` keeps anon INSERT. | Controlled | Recheck grants after every new public table migration. |
| Chatbot handoff stores personal data         | Chatbot / privacy        | Handoff is consent-based, honeypot-protected and stored server-side only.         | Controlled | Browser QA the form and keep privacy/terms acknowledgement required. |
| Production data changed accidentally         | Supabase / Netlify       | Local-only execution rule; migrations not applied.                               | Controlled | Require explicit approval before any production action.                                                                              |
| Admin lead data indexed or linked publicly   | Admin-lite               | Admin metadata noindex/nofollow; sitemap/nav/footer exclude admin.               | Controlled | Recheck before deployment.                                                                                                           |
| Service role key exposed to browser          | Lead storage             | Service role usage is server-side only and not `NEXT_PUBLIC`.                    | Controlled | Check build output before release.                                                                                                   |
| Private rates or margins exposed             | Estimate / copy          | Private rate card remains server-side; public copy uses ranges only; public safety scan is available. | Controlled | Run `npm run qa:public-safety` before release. |
| Legal or compliance overclaim                | Public content / chatbot | Copy says planning guidance only and not legal advice.                           | Controlled | Review new content through QA / Anti-Leak Agent.                                                                                     |
| Uploaded quote files exposed                 | Quote review             | Upload UI is placeholder-safe; public API leak tests reject fake storage URL/path exposure; private storage decision brief and policy design exist. | Partial    | Review the policy design, then create a separate SQL-only/local-migration packet before any implementation. |
| Notification provider failure blocks leads   | Notifications            | Lead storage and notification preparation are decoupled; public API tests cover provider failure without key leaks. | Controlled | Add real staging email send QA when env vars exist.                                                                                  |
| Admin token missing in local QA              | Admin-lite               | Admin APIs reject unauthenticated access; no data exposure.                      | Controlled | Set `OPERON_BATHROOMS_ADMIN_TOKEN` for authorised admin QA.                                                                          |
| Dev server manifest corrupted after build    | Local QA                 | Restarting `next dev` before server crawls clears the issue.                     | Controlled | Do not run server-based crawls against a dev process that stayed open during `next build`.                                           |
| Chatbot gives unsafe pricing advice          | Chatbot                  | Rule-based responses refuse binding pricing and private rates.                   | Controlled | Keep tests for risky intents current.                                                                                                |
| Thin local SEO pages created too early       | SEO                      | Areas page is a hub only; no suburb page generation.                             | Controlled | Add local pages only with unique content.                                                                                            |
